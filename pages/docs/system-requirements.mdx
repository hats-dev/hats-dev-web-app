# System Requirements

Install the following programs to configure your dev environment.

Programs marked with an asterik **\*** are **required** for HaTs to work.

## Node

- <a target="_blank" href="https://www.npmjs.com/package/n">
  	Node (v11.14 or higher)
  </a>
  *

## Git + GitHub CLI

- <a
  	target="_blank"
  	href="https://git-scm.com/book/en/v2/Getting-Started-Installing-Git">
  	Git (v2.3 or higher)
  </a>
  *
- <a target="_blank" href="https://cli.github.com/manual">
  	GitHub CLI (latest stable version is recommended)
  </a>
  *

One of the core benefits you receive each time you bootstrap a new project with `hats create` is automatic synchronization between your local repository and a fresh GitHub remote. Under the hood, HaTs leverages the GitHub CLI to facilitate this tethering.

> **Note**: The GitHub CLI **must** have permission to create repos for the GitHub account or organization that will own the remote, for instance via an SSH key. A tutorial for configuring SSH access to GitHub is <a target="_blank" href="https://docs.github.com/en/authentication/connecting-to-github-with-ssh/adding-a-new-ssh-key-to-your-github-account">linked here</a>.

### Verify your GitHub CLI Configuration

1. <u>Verify the GitHub CLI version</u>

```bash
$ gh version
gh version 2.6.0 (2022-03-15)
https://github.com/cli/cli/releases/tag/v2.6.0
```

2. <u>Login to GitHub and verify your Auth status (SSH Example)</u>

```bash
$ gh auth status
github.com
  ✓ Logged in to github.com as <gh-username> (/Users/<shell-username>/.config/gh/hosts.yml)
  ✓ Git operations for github.com configured to use ssh protocol.
  ✓ Token: *******************
```

```bash
$ gh ssh-key list
GitHub CLI                    ssh-ed25519 AAAAC...****************  15d
```
